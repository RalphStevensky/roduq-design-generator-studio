## Layout patterns

### Hero
- **Personal statement**: "{Role} z {city}. {Specialty}."
- **Split with photo**: portrait + text
- **Work-led**: latest project as hero tile
- **Statement design**: experimental layout (designers showing skills through layout)

### Work grid
- **Uniform grid**: 2-3 col, equal aspect ratios (clean, scannable)
- **Masonry**: uneven aspects (dla photographers, illustrators)
- **Horizontal scroll**: dla < 6 pieces (less common but distinctive)
- **List with thumbnails**: dla content-heavy creators (writers, podcasters)

### Project detail pages
- **Full-bleed images** + minimal text dla visual work
- **Case study format** (story-led) dla UX / branding
- **Code snippets** dla developer portfolios
- **Process video** dla animators / motion designers

### About
- **Split photo + text**
- **Story-led** (long-form essay)
- **Stats prominent** (years / clients / projects)

### Skills / Tools
- **Tag cloud** dla generalists
- **Categorized list** (e.g. "Design tools" / "Development" / "Other")
- **Tool icons** dla developers (Figma logo / VS Code / Git icons)
- ❌ NIE używaj skill bars ("Photoshop 90%") — outdated, looks tacky

## Anti-patterns

❌ **Generic stock photos** — credibility killer
❌ **Skill bars** ("HTML 95%") — looks dated, evidences nothing
❌ **No personality** — corporate copy w personal site is weird
❌ **No availability signal** — clients want to know
❌ **Hidden contact info** — make it easy to hire
❌ **Long bio paragraph blocks** w hero — keep hero focused
✅ **Real photo of creator** (informal preferred)
✅ **Available / Booked status** w hero or footer
✅ **Direct contact: email + form** (NIE just form)
✅ **Show work first, talk about self second**
